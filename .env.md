# .env Setup Guide - Easy Development with Security

## Overview

The system now uses **.env files** instead of system environment variables for easier local development, while maintaining the **same security level**:

- **`.env.example`** - Template file (safe for GitHub) ✅ Commits to git
- **`.env`** - Your actual secrets (local copy) ❌ Never commits (in .gitignore)
- **`config.json`** - Uses `${ENV_VAR}` placeholders (safe for GitHub) ✅ Commits to git

---

## Quick Start (3 Steps)

### 1. Edit your `.env` file with actual secrets

```bash
# File: .env (in project root)
ENCRYPTION_PASSPHRASE=my-strong-passphrase-12345
RECIPIENT_PRIVATE_KEY_PATH=keys/recipient_kyber768_private.key
SENDER_PRIVATE_KEY_PATH=keys/sender_dilithium3_private.key
```

### 2. Install python-dotenv

```bash
pip install python-dotenv
```

### 3. Run your system

```bash
python main.py
```

**That's it!** The `.env` file is automatically loaded and environment variables are substituted into `config.json`.

## How It Works

```
┌─────────────────────────────────────────────────────┐
│  User runs: python main.py                          │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│  config_loader_secure.py:                           │
│  1. load_env_file('.env')                           │
│     └→ Loads environment variables from .env        │
│  2. load config.json                                │
│  3. substitute_environment_variables()              │
│     └→ Replaces ${ENCRYPTION_PASSPHRASE}            │
│        with actual value from .env                  │
└─────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────┐
│  config dictionary (fully resolved):                │
│  {                                                  │
│    "key_protection": {                              │
│      "passphrase": "my-strong-passphrase-12345"    │
│    }                                                │
│  }                                                  │
└─────────────────────────────────────────────────────┘
```

```
│   │   Contains: ${ENV_VAR} placeholders
│   │

│       Contains: ${ENV_VAR} placeholders
│       (substituted from .env at runtime)
│
├── .gitignore ✅ (updated)
│   Contains:
│   - .env (never commit your secrets!)
│   - config/config.json
│   - keys/
│   - output/
│
└── utils/
    └── config_loader_secure.py ✅ (loads .env)
```

## Environment Variables

### Required

| Variable | Purpose | Example |
|----------|---------|---------|
| `ENCRYPTION_PASSPHRASE` | Scrypt key for protecting keys at rest | `my-strong-passphrase-12345` |
| `RECIPIENT_PRIVATE_KEY_PATH` | Kyber768 private key for decryption | `keys/recipient_kyber768_private.key` |
| `SENDER_PRIVATE_KEY_PATH` | Dilithium3 private key for signing | `keys/sender_dilithium3_private.key` |

### All values from .env are automatically substituted

```python
# config.json (template with ${ENV_VAR} placeholders)
{
  "key_protection": {
    "passphrase": "${ENCRYPTION_PASSPHRASE}"
  },
  "post_quantum": {
    "recipient_private_key_path": "${RECIPIENT_PRIVATE_KEY_PATH}"
  },
  "metadata_signature": {
    "sender_private_key_path": "${SENDER_PRIVATE_KEY_PATH}"
  }
}

# At runtime, becomes:
{
  "key_protection": {
    "passphrase": "my-strong-passphrase-12345"
  },
  "post_quantum": {
    "recipient_private_key_path": "keys/recipient_kyber768_private.key"
  },
  "metadata_signature": {
    "sender_private_key_path": "keys/sender_dilithium3_private.key"
  }
}
```

## Security (Same Level as System Env Vars)

✅ **Secrets NOT stored in config.json**
- Plaintext secrets only in local .env file
- .env is in .gitignore (never commits)
- Attackers need access to your local machine

✅ **File permissions still enforced**
- Startup checks verify 0o600 permissions on config files
- Warnings logged if permissions too permissive

✅ **No secrets in logs**
- Config structure logged, not actual values
- Environment variable names logged, not values

✅ **Backward compatible**
- Old plaintext configs still work (with warnings)
- Smooth migration path

## Usage Examples

### Windows PowerShell

```powershell
# Edit .env in your favorite editor
notepad .env

# Then run:
python main.py
```

### Linux/macOS

```bash
# Edit .env
nano .env

# Then run:
python main.py
```

### Docker

```dockerfile
FROM python:3.10

WORKDIR /app
COPY . .

# Copy .env.example as reference (not the actual .env!)
COPY .env.example .env.example

# Set environment variables from secrets
ENV ENCRYPTION_PASSPHRASE="docker-secret-passphrase"
ENV RECIPIENT_PRIVATE_KEY_PATH="/app/keys/recipient_kyber768_private.key"
ENV SENDER_PRIVATE_KEY_PATH="/app/keys/sender_dilithium3_private.key"

RUN pip install -r requirements.txt

# Run the system
CMD ["python", "main.py"]
```

### CI/CD (GitHub Actions)

```yaml
jobs:
  encrypt:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Set environment from secrets
        run: |
          cat > .env << EOF
          ENCRYPTION_PASSPHRASE=${{ secrets.ENCRYPTION_PASSPHRASE }}
          RECIPIENT_PRIVATE_KEY_PATH=${{ secrets.RECIPIENT_PRIVATE_KEY_PATH }}
          SENDER_PRIVATE_KEY_PATH=${{ secrets.SENDER_PRIVATE_KEY_PATH }}
          EOF
      
      - name: Run encryption
        run: python main.py --mode encrypt --input test.png
```

## Troubleshooting

### Error: "Module dotenv not found"

**Solution:** Install python-dotenv
```bash
pip install python-dotenv
```

### Error: "ENCRYPTION_PASSPHRASE not set"

**Solution:** Make sure .env file exists and contains:
```bash
ENCRYPTION_PASSPHRASE=your-actual-passphrase
```

**Verify .env is being loaded:**
```python
from dotenv import load_dotenv
import os
load_dotenv()
print(os.getenv('ENCRYPTION_PASSPHRASE'))  # Should print your passphrase
```

### .env file changes not taking effect

**Solution:** Some IDEs cache env vars. Restart Python:
```bash
# Kill any running Python processes
# Then run again:
python main.py
```

### I accidentally committed .env to git!

**Solution:** Rotate all secrets immediately:
```bash
# Remove from git history
git rm --cached .env
git commit --amend

# Create new .env with different secrets
rm .env
python setup_env.py
# Edit .env with NEW secrets
```

### What Changed

| Component | Before | After |
|-----------|--------|-------|
| Env vars | System env vars (export/set) | `.env` file (python-dotenv) |
| Setup complexity | Multi-step per platform | Single .env file |
| Development speed | Slower (env var setup) | Faster (edit .env, run) |
| Security | Same | Same ✅ |
| Commits | System env not tracked | .env.example as template |
| Portability | OS-specific | Unified approach |

- ✅ `.gitignore` - Already blocks `.env`
